PLOT 후불제 소개팅 앱 (프리랜서)

프로젝트 개요

PLOT은 사용자가 실제 매칭이 이루어진 경우에만 결제가 발생하고 불필요한 비용 부담 없이 성사된 만남에만 비용을 지불하는 합리적인 후불제 소개팅 앱입니다.

성능 개선

소개 가능 횟수 조회 성능 최적화
문제상황
  1. 유저 N명에 대해 개별 반복 호출로 N회 DB 조회 발생
  2. 각 유저의 관련 엔티티 조회 시 N+1 쿼리 문제 발생
해결방법
  1. 유저 조건 배치를 IN절 Batch Query로 통합 조회
  2. 연관 엔티티를 Fetch Join으로 한번에 조회
성과
  1. 쿼리 N회 → 1회로 통합 (소개 가능 한사람을 한번에 호출)
  2. N+1 쿼리 제거
  3. DB 부하 감소 및 응답속도 90% 이상 개선
로그/모니터링 고도화
  • 개선사항: Logbook, Logback, MDC 활용한 구조화된 로그 시스템
  • 기능: API 호출별 상세 로그, 요청/응답 바디 자동 마스킹
  • 동시성 문제 해결: MDC를 활용한 멀티스레드 환경에서 꼬이거나 누락되는 동시성 문제를 해결하고 로그 추적성을 확보
  • 개인정보 보호: 이름, 휴대폰, 상세주소 등 필드별 정규식 마스킹
  • 성과: DataDog에서 TraceID 기반 검색 및 필터링 개선, 모든 로그 누락 없이 안정적으로 수집, 보안 감사 대응력 향상
테스트 환경 최적화
  • 문제상황: OutOfMemoryError 발생, 테스트 시간 9분 20초 소요
  • 원인: 과도한 Bean 초기화와 병렬화 미흡
  • 해결방법: 힙/메타스페이스 한도 명시, Spring Lazy Initialization, 병렬 처리
  • 성과: OOME 해결, 테스트 시간 9분 20초 → 6분 40초로 단축 (약 28% 단축)
